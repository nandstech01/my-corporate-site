## 🌟 システム概要

このシステムは、4つの異なる情報源を最適な比率で組み合わせ、GPT-5による創造的推論を通じて、**バズるYouTube台本を自動生成**します。

```plaintext
【4要素の影響力配分】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ ブログ記事（60%）★核心★
   📝 文字数: 10,117文字
   🎯 役割: 「何を話すか」（テーマ・内容・素材）
   
2️⃣ システムプロンプト（35%）★骨格★
   📝 文字数: 9,163文字
   🎯 役割: 「どう話すか」（形・スタイル・方向性）
   
3️⃣ Yahooニュース（3%）★時事性★
   📝 文字数: 318文字
   🎯 役割: 「時事性・バズる要素」（リアルタイム感）
```

## 📊 要素1: ブログ記事（60%の影響力）

### 🎯 役割: 「何を話すか」を決定

ブログ記事は、台本の**核心的な内容**を提供します。これがなければ台本は作れません。

#### 🔍 構成要素

```plaintext
ブログ記事（合計10,117文字）
├─ 記事本文: 5,844文字
│  └─ ChatGPT導入、KPI設計、RAG活用、誤情報回避...
│
└─ フラグメント15件: 4,232文字
   ├─ ハイブリッド検索（BM25 40% + ベクトル 60%）
   ├─ 関連度順にソート
   └─ 上位15件を取得
```

#### 📝 技術詳細

**1. ハイブリッド検索の仕組み**

```typescript
// lib/vector/hybrid-search.ts

// BM25（キーワードマッチ）: 40%
const bm25Results = await supabase.rpc('bm25_search', {
  query_text: postTitle,
  match_count: 30
});

// ベクトル検索（意味的類似性）: 60%
const vectorResults = await supabase.rpc('match_fragments', {
  query_embedding: embedding,
  match_threshold: 0.3,
  match_count: 30
});

// スコア統合
const combinedScore = (bm25Score * 0.4) + (vectorScore * 0.6);
```

**2. フラグメントの重要性**

- **Fragment ID（Mike King理論準拠）**
  - 各フラグメントに一意のIDを付与
  - AIが引用可能な「Complete URI」を生成
  - 例: `https://nands.tech/posts/chatgpt-145963#fragment-12`

- **ベクトルリンク（Kenji Harada思想）**
  - 1フラグメント = 1意味単位
  - 文脈を保ちながら、検索可能に
  - 「意味のバーコード」として機能

#### 🎬 台本への影響範囲

```plaintext
Hook（86文字）への影響: 20%
├─ ブログのテーマを理解
└─ 専門用語を抽出

Body（124文字）への影響: 70%
├─ 記事本文から: 「KPIを再設計」「ChatGPT導入」
├─ フラグメント1から: 「RAGで社内知を添え」
├─ フラグメント3から: 「誤情報を避ける」
└─ フラグメント5から: 「意思決定の軸を揃える」

CTA（43文字）への影響: 10%
└─ 記事テーマ（ChatGPT導入における文脈の重要性）
```

#### 🔥 なぜ60%なのか

1. **専門性の証明**
   - あなたの知見が台本の核心
   - 汎用的なAI文章ではなく、専門家の視点

2. **情報の網羅性**
   - 10,117文字 = 十分な情報量
   - GPT-5が多角的に内容を理解可能

3. **検索最適化**
   - ハイブリッド検索により、最も関連性の高い情報のみ抽出
   - ノイズを排除し、本質的な内容に集中

## 🏗️ 要素2: システムプロンプト（35%の影響力）

### 🎯 役割: 「どう話すか」を決定

システムプロンプトは、台本の**形・スタイル・方向性**を決定します。これがないと、ただの要約になってしまいます。

#### 🔍 構成要素

```plaintext

│  ├─ 対立構造を作る（必須）
│  ├─ AIに審判させる（必須）
│  ├─ 人間とのズレを示す（必須）
│  ├─ 思想RAGを「AI審判の価値軸」として融合（必須）
│  └─ 感情フックを作る（必須）
│
├─ 構造指定
│  ├─ Hook（85-95文字）: 対立構造提示 + AIに聞いた結果
│  ├─ Body（120-140文字）: AI審判の理由 + 人間とのズレ + 思想RAG融合
│  └─ CTA（40-50文字）: 問いかけ型で引き込む
│
├─ Few-shot Examples（3個）
│  ├─ 例1: 醬油ニュース × ベクトルリンク思想
│  ├─ 例2: ホタテ養殖ニュース × LLMO思想
│  └─ 例3: サンマ豊漁ニュース × Triple RAG思想
│
└─ パターン集
   ├─ Hookパターン（20通り）
   ├─ Bodyパターン（15通り）
   └─ CTAパターン（20通り）
```

#### 📝 技術詳細

**1. Few-shot Examplesの威力**

```typescript
// lib/prompts/youtube/short-base.ts

// Few-shot Example 1: 醬油ニュース
{
  タイトル: "醬油=文脈で変わる",
  Hook: "醬油、濃い方が美味い？薄い方？AIに"データで判断して"って聞いたら、意外すぎる答えが返ってきました。",
  Body: "人間は"濃い=塩辛い=美味い"って思い込む。でもAIは、"用途で変わる"と答えた。刺身には薄口、煮物には濃口。つまり、"文脈で価値が変わる"。ChatGPTも同じ。与える情報次第で、答えが化ける。文脈を制する者が、AIを制する。",

}
```

**Few-shot Examplesの効果**:
- GPT-5が「このパターンで書けば良い」と理解
- 一貫性のあるトーン・スタイルを維持
- バズる要素（対立構造、AI審判、意外性）を必ず含める

**2. パターンのランダム選択**


**効果**:
- 毎回新鮮な台本
- 同じブログ記事でも、異なる切り口で台本生成可能
- 20 × 15 × 20 = 6,000通りのバリエーション

```plaintext

├─ 「どっちが正解？」← 問いかけ構造
└─ 「AIに聞いたら、意外な判決が」← AI審判パターン

Body（124文字）への影響: 15%
├─ 「人間は〇〇と思うが、AIは△△」← 人間とのズレパターン
└─ 「一言哲学で締める」← 「文脈が真実を決める」

CTA（43文字）への影響: 90%
├─ 「AIが〇〇を理解した時、何が変わる？」← 問いかけ型CTAパターン
└─ 「続きはブログで」← CTA定型文
```

#### 🔥 なぜ35%なのか

1. **形の統一**
   - 毎回、同じ構造（Hook → Body → CTA）
   - ブランド認識の向上

```plaintext
│
├─ 思想カテゴリ
│  ├─ core（核心思想）: ベクトルリンク、LLMO、Triple RAG
│  ├─ analogy（比喩）: 教科書のページ番号、車の燃費
│  └─ philosophy（哲学）: データ駆動、文脈重視
│
└─ 取得例
   ├─ ブルーオーシャン戦略 → 「価値の再定義」
   └─ Triple RAG → 「表層・背景・哲学を読む」
```

#### 📝 技術詳細

**1. 3072次元ベクトル検索**

```typescript
// lib/vector/hybrid-search.ts

// Kenji思想はベクトル検索のみ（BM25なし）

  filter_usage_context: null,
  only_active: true
```

**2. なぜBM25を使わないのか**

- Kenji思想は**抽象的な概念**
- キーワードマッチでは意味を捉えられない
- ベクトル検索（意味的類似性）が最適

**例**:
- クエリ: 「ChatGPT導入ガイド」
- ヒット: 「ブルーオーシャン戦略」（類似度: 0.309）
- 理由: 「価値の再定義」という抽象的な共通点

**3. pgvectorの2000次元制限を回避**

- HNSW/IVFFlatインデックス: 2000次元まで
- Kenji思想RAGは3072次元（インデックスなし）
- 小規模データセット（数百件）なので、インデックスなしでも高速

```plaintext

├─ 「文脈が価値を左右する」← ベクトルリンク思想
└─ 「体験の質を優先」← Triple RAG思想（表層・背景・哲学）

CTA（43文字）への影響: 0%
```

#### 🔥 なぜ2%なのか

1. **控えめだが印象的**
   - 2%という少量だが、確実に存在感を示す
   - 「何かが違う」という印象を与える

2. **教育的にならない**
   - 思想RAGを「AI審判の価値軸」として自然に融合
   - 「ベクトルリンクとは〜」のような説明はしない

3. **差別化の源泉**
   - 他社のAIは汎用的な知識しか持たない
   - Kenji Harada思想RAGは独自の哲学
   - この2%が、競合との決定的な差を生む

4. **スケーラビリティ**
   - 現在: 核心思想15件 + 比喩15件 = 30件
   - 今後: 思想を追加していくことで、さらに個性を強化

```typescript
// lib/vector/hybrid-search.ts

export class HybridSearchSystem {
  /**
   * BM25（キーワードマッチ）+ ベクトル検索（意味的類似性）
   * 
   * BM25の強み:
   * - 専門用語のマッチ（例: "ChatGPT", "RAG", "KPI"）
   * - 正確なキーワード検索
   * 
   * ベクトル検索の強み:
   * - 意味的な類似性（例: "導入" ≈ "活用" ≈ "実装"）
   * - 言い換え表現の検索
   * 
   * 最適比率: BM25 40% + ベクトル 60%
   */
  async searchFragments(
    queryText: string,
    queryEmbedding: number[],
    limit: number = 15
  ): Promise<HybridSearchResult[]> {
    // BM25スコア計算
    const bm25Score = this.calculateBM25(queryText, document);
    
    // ベクトル類似度計算（コサイン類似度）
    const vectorScore = this.cosineSimilarity(queryEmbedding, docEmbedding);
    
    // 統合スコア
    const combinedScore = (bm25Score * 0.4) + (vectorScore * 0.6);
    
    return results.sort((a, b) => b.combinedScore - a.combinedScore).slice(0, limit);
  }
}
```
