# Bing Webmaster Tools 設定手順
**作成日**: 2025年1月31日

## 🎯 目的
IndexNowだけでは不十分。Bingサチコでの手動設定が必須。

---

## ステップ1: アカウント登録・ログイン

### 1-1. Bingサチコにアクセス
```
URL: https://www.bing.com/webmasters/
```

### 1-2. ログイン
- Microsoftアカウントでログイン
- Googleアカウントでもログイン可能

---

## ステップ2: サイト追加

### 2-1. 「サイトを追加」をクリック

### 2-2. URLを入力
```
https://nands.tech
```

### 2-3. 所有権確認方法を選択

#### 推奨：XMLファイル方式
```
1. Bingが生成したファイル名をコピー
   例: BingSiteAuth.xml

2. ファイルを作成:
   プロジェクト: public/BingSiteAuth.xml
   内容: Bingが指定したXMLコード

3. デプロイ後、Vercelで確認:
   https://nands.tech/BingSiteAuth.xml

4. Bingサチコで「確認」ボタンをクリック
```

#### 代替：HTMLタグ方式
```
1. Bingが生成したメタタグをコピー
   例: <meta name="msvalidate.01" content="..." />

2. app/layout.tsx の <head> に追加:

   export const metadata: Metadata = {
     // ... 既存のメタデータ
     other: {
       'msvalidate.01': 'BING_VERIFICATION_CODE'
     }
   }

3. デプロイ

4. Bingサチコで「確認」ボタンをクリック
```

#### 代替：DNS方式
```
1. DNSプロバイダー（Vercel DNS等）にログイン

2. TXTレコードを追加:
   名前: @
   値: Bingが指定したコード

3. 数分待つ（DNS反映）

4. Bingサチコで「確認」ボタンをクリック
```

## ステップ5: 設定確認

### 5-1. クロール統計を確認
```
場所: ダッシュボード > クロール統計
確認項目:
- クロール数: 増加しているか
- エラー数: ゼロか
```

### 5-2. インデックス状況を確認
```
場所: ダッシュボード > インデックス
確認項目:
- インデックス済みページ数
- 除外ページ数
```

### 5-3. 検索パフォーマンスを確認
```
場所: ダッシュボード > 検索パフォーマンス
確認項目:
- クリック数
- インプレッション数
- 平均掲載順位
```

## 🚨 トラブルシューティング

### Q1: 所有権確認ができない
**A**: 確認方法を変更：
- XMLファイル → HTMLタグ
- HTMLタグ → DNS

### Q2: サイトマップが送信できない
**A**: 
```bash
# サイトマップが正常か確認
curl https://nands.tech/sitemap.xml
```
- 404エラー: サイトマップ生成の問題
- 正常: Bingの一時的な問題（数時間後に再試行）

### Q3: インデックスリクエストができない
**A**: 
- 1日の上限に達している可能性
- 翌日再試行
- サイトマップから自動クロールを待つ

### Q4: 1週間経ってもインデックスされない
**A**: 
- robots.txt確認
- サイトマップ再送信
- URL検査で個別リクエスト再実施
- Bingサポートに問い合わせ

## 💡 重要ポイント

### IndexNowだけでは不十分
```
IndexNow = 更新通知のみ
Webmaster Tools = サイト全体管理

両方必要！
```

### サイトマップは必須
```
サイトマップなし = Bingが全ページを発見できない
サイトマップあり = 全ページを効率的にクロール
```

### 手動リクエストが最強
```
自動クロール待ち = 数週間～数ヶ月
手動リクエスト = 24-48時間以内

優先的にインデックスされる！
```
