## 📊 Before/After 比較

### Before: 純粋ベクトル検索のみ

```typescript
// 検索プロセス
1. クエリをベクトル化
2. コサイン類似度で検索
3. 類似度でソート

// 問題点
❌ キーワード完全一致が弱い
   - 例: "ChatGPT"という固有名詞の完全一致が難しい
   - 類似概念（AI、GPT、言語モデル）も混ざる

❌ 最新情報の優先度が不明確
   - 古い情報と新しい情報が同じ類似度なら区別できない

❌ 意味は似ているが文脈が違うものも取得
   - 例: "AIエージェント開発"と"AI活用事例"が同じスコア
```

### After: ハイブリッド検索（BM25 + ベクトル + RRF）

```typescript
// 検索プロセス
1. 【BM25全文検索】
   - キーワード完全一致を評価
   - tf-idf + 文書長正規化
   - 固有名詞に強い

2. 【ベクトル類似度検索】
   - 意味的類似性を評価
   - コサイン距離
   - 同義語・関連概念に強い

3. 【RRF統合】
   - 両方の検索結果をランク融合
   - combined_score = 0.4 * BM25 + 0.6 * Vector
   - Trend RAGは鮮度スコア追加（+0.3 * Recency）

// 改善点
✅ キーワード完全一致の精度UP
   - 例: "ChatGPT"を含む記事を確実に上位に

✅ 意味的類似性も維持
   - 例: "GPT-4", "大規模言語モデル"も取得

✅ 最新情報を優先（Trend RAG）
   - 24時間以内の情報を最優先
   - 7日間で線形減衰

✅ 総合的な関連性向上
   - キーワード + 意味 + 鮮度の3軸評価
```

## ⚠️ 現在の問題（Company RAG）

### 症状
- データベース関数は正常動作（BM25スコア取得確認）
- HybridSearchSystem経由だと0件

### 調査結果
```sql
-- ✅ データベース直接実行: 動作OK
SELECT * FROM hybrid_search_company_vectors(
  'ChatGPT',
  (SELECT embedding FROM company_vectors LIMIT 1),
  0.1, 3, 0.4, 0.6
);
-- 結果: 3件取得、BM25スコア=0.1確認

-- ❌ APIレイヤー経由: 0件
curl -X POST /api/test-hybrid-search 
  -d '{"query":"ChatGPT","source":"company","limit":3}'
-- 結果: 0件
```

### 推定原因
1. **ベクトル化の問題**
   - OpenAIEmbeddings.embedSingle()が正しく動作していない可能性

2. **Supabase RPC呼び出しの問題**
   - パラメータの型変換エラー
   - vector型の渡し方が不正

3. **キャッシュの問題**
   - 開発サーバーのキャッシュが古い
