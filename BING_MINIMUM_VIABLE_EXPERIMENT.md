## 📋 実験フェーズ1: 問題の特定（11月5日）

### 1-1. Bingサチコで「除外47」の詳細を確認

**実施場所**: サイトエクスプローラ > すべてのURL > 「除外」タブ

**確認項目チェックリスト**:
```
□ 除外されているURLのリスト
□ 各URLの除外理由
  - 重複コンテンツ
  - noindexタグ
  - robots.txtブロック
  - 低品質・薄い内容
  - クロール済み - インデックス未登録
  - 発見済み - 未クロール
□ 除外された日付
```

### 1-2. 除外理由の分類

**想定される分類**:
```
【分類A】技術的問題（即修正可能）
- noindexタグが設定されている
- robots.txtでブロックされている
- canonical設定の不備

【分類B】コンテンツ品質問題（改善必要）
- 重複コンテンツ判定
- 低品質・薄い内容
- 内部リンク不足

【分類C】クロール待ち（待機必要）
- 発見済み - 未クロール
- 新しいページ（認識まで時間が必要）
```

### 1-3. 実験対象ページの選定

**選定基準**:
```
優先度1: 重要度が高い × 修正が容易
✅ /aio-seo
✅ /system-development
✅ /corporate
✅ /ai-agents
✅ /vector-rag

優先度2: トラフィックポテンシャルが高い
✅ /chatbot-development
✅ /hr-solutions
✅ /mcp-servers

除外: すでにインプレッションがあるページ
❌ / (トップページ) → 既に156インプレッション
```

**最終選定**: `/aio-seo`（理由: 最も重要＋コンテンツ充実＋差別化明確）

### シナリオC: 失敗（クロールもされない）

**判断基準**:
```
❌ 7日経過してもクロールされない
❌ URL検査で「発見済み - 未クロール」のまま
```

**原因究明**:
```
1. robots.txt再確認
   curl https://nands.tech/robots.txt | grep aio-seo
   → Disallow されていないか確認

2. サイトマップ再確認
   curl https://nands.tech/sitemap.xml | grep aio-seo
   → 含まれているか確認

3. サーバーレスポンス確認
   curl -I https://nands.tech/aio-seo
   → HTTP 200が返ってくるか確認

4. Bingクローラーのブロック確認
   - User-agent: Bingbot がブロックされていないか
   - Vercel設定でブロックされていないか
```

**対処法**:
```
1. サイトマップ再送信
2. IndexNow再送信（1日おきに3回まで）
3. Bing Webmaster Tools > ヘルプ > サポートに問い合わせ
4. 代替ページで実験（/system-development等）
```

## 🎓 学びと仮説

### 仮説1: Bingは重複排除が強い
```
根拠: 提示された意見「Googleより重複排除アルゴリズムが強い」

検証方法:
- canonical明示で除外解除されるか観察
- 同じテンプレート構造のページ群の挙動を比較
```

### 仮説2: 手動リクエストが最も効果的
```
根拠: トップページは156インプレッション（唯一の成功例）

検証方法:
- 手動リクエストしたページのインデックス速度を測定
- IndexNowのみ vs 手動リクエスト併用 の比較
```

### 仮説3: 内部リンクがクロール頻度を上げる
```
根拠: SEO一般論

検証方法:
- トップページからリンク追加前後でクロール頻度比較
- サイトマップのみ vs 内部リンクあり の比較
```
