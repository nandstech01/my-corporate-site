# 🚨 緊急：IndexNowキー404エラー修正

**発見日**: 2025年1月31日  
**問題**: 古いIndexNowキーファイルが404エラー

---

## 🔍 問題の詳細

### Googleサーチコンソールで発見
```
URL: 9c8eed8b08354c248708db16c4abd17
HTTPコード: 404
最終クロール: 14 9月 2025
```

### 現在の正しいキー
```
正しいキー: b247e7b751dc4d84164c134151ee0814
ファイル: public/b247e7b751dc4d84164c134151ee0814.txt
ステータス: ✅ 正常（確認済み）
```

---

## 🎯 原因分析

### 可能性1: Vercel環境変数の設定ミス
Vercelの環境変数 `INDEXNOW_KEY` に古い値が設定されている可能性

**確認方法**:
1. Vercelダッシュボード > プロジェクト設定
2. Environment Variables
3. `INDEXNOW_KEY` の値を確認

**期待値**: `b247e7b751dc4d84164c134151ee0814`  
**実際の値**: `9c8eed8b08354c248708db16c4abd17` ← これなら修正が必要

### 可能性2: 過去のデプロイで使われていた
過去に `9c8eed8b08354c248708db16c4abd17` を使っていた時期があり、
Googleがそのキーを記憶している

**対策**: 古いキーファイルを追加で配置する

---

## ✅ 修正手順（優先度順）

### 手順1: Vercel環境変数の確認・修正
1. Vercelダッシュボードにログイン
2. プロジェクト（my-corporate-site）を選択
3. Settings > Environment Variables
4. `INDEXNOW_KEY` の値を確認：
   - 現在の値: ________
   - 正しい値: `b247e7b751dc4d84164c134151ee0814`
5. 違う場合は修正して保存
6. プロジェクトを再デプロイ

### 手順2: 古いキーファイルを追加配置（暫定対応）
古いキーに対する404エラーを解消するため、古いキーファイルも配置：

```bash
# 古いキーファイルを作成
echo "9c8eed8b08354c248708db16c4abd17" > public/9c8eed8b08354c248708db16c4abd17.txt
```

これにより：
- 新しいキー: `b247e7b751dc4d84164c134151ee0814.txt` ✅
- 古いキー: `9c8eed8b08354c248708db16c4abd17.txt` ✅（暫定）

### 手順3: 両方のキーでIndexNow送信
新しいキーと古いキーの両方でBingに通知：

```bash
# 新しいキーで送信（現在のAPI）
curl https://nands.tech/api/indexnow/submit-all

# 古いキーでも送信（念のため）
curl -X POST https://api.indexnow.org/indexnow \
  -H "Content-Type: application/json" \
  -d '{
    "host": "nands.tech",
    "key": "9c8eed8b08354c248708db16c4abd17",
    "keyLocation": "https://nands.tech/9c8eed8b08354c248708db16c4abd17.txt",
    "urlList": ["https://nands.tech/"]
  }'
```

### 手順4: Googleサーチコンソールでの対応
1. Google Search Console > カバレッジ
2. 404エラーURLを確認
3. 修正後、「修正を検証」をクリック
4. 再クロールをリクエスト

---

## 📊 「除外: 59ページ」問題の対応

### 除外理由の確認
Google Search Console > カバレッジ > 除外タブ

**よくある除外理由**:
- noindex タグ
- robots.txt でブロック
- 重複コンテンツ
- 低品質コンテンツ
- クロール済み - インデックス未登録

### 対策
1. 除外されているURLリストをエクスポート
2. 各URLの除外理由を確認
3. 主要ページが除外されている場合は修正：
   - noindex削除
   - canonical URL設定
   - コンテンツ品質向上
   - 内部リンク強化

## 📚 参考情報

### IndexNow仕様
- 1つのドメインに複数のキーを使用可能
- キーファイルは永続的にアクセス可能である必要がある
- キー変更後も古いキーを一定期間残すことを推奨

### Google除外理由
- 公式ドキュメント: https://support.google.com/webmasters/answer/7440203
- 除外は必ずしも悪いことではない（意図的な場合もある）
- 主要ページが除外されている場合は要対応
