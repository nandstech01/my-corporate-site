## 🌟 システム概要

このシステムは、**6つの異なる情報源**を最適な比率で組み合わせ、GPT-5による創造的推論を通じて、**唯一無二の中尺動画台本（900-1000文字）**を自動生成します。

**最大の特徴**: 
- Kenji Haradaの「静かなる構築者」という個人ストーリーを全体のトーンとして統一
- **CapCut AIアバター生成で自然に喋らせられる品質**を実現
- **THINK-RAG**で「人間らしい思考の息づかい」を付与

## 📐 台本構成: 5つのフェーズ

### 1️⃣ Hook（150文字）: 問題提起で視聴者を引き込む

```plaintext
【目的】
- 視聴者が共感できる問題を提示
- 客観的な事実を提示（個人的な主張・実績アピールはNG）

【例】
「ある企業で、ChatGPTのUIだけを導入しました。
 でも、成果は出なかった。
 原因は明確です。ChatGPTは社内の事実を知りません。
 構造で橋を架ける必要があります。
 今日は、その構造についてお話しします。」

【❌ 避ける表現】
- 「私は〜してきました」（実績アピール）
- 「構造を信じてきました」（偉そう）

【✅ 推奨表現】
- 「今日は、その構造についてお話しします」（完結、自然、謙虚）
```

### 2️⃣ Body1（250文字）: ノウハウの全体像を示す

```plaintext
【目的】
- ブログ記事の核心を抽出
- 「まず理解すべきなのは〜」で始める
- 客観的な情報提供に徹する

【例】
「まず理解すべきなのは、この技術の構造です。
 複雑に見えますが、実は単純なんです。
 鍵は三つあります。
 一つ目はユースケースの選定、
 二つ目はデータ連携、
 三つ目は運用設計です。
 これだけで動きます。」

【❌ 避ける表現】
- 「鍵は三つ。」（断片的、音声で不自然）

【✅ 推奨表現】
- 「鍵は三つあります。一つ目は〜、二つ目は〜、三つ目は〜です。」（完結、自然）
```

### 3️⃣ Twist（200文字）: 哲学的洞察 - THINK-RAG統合

```plaintext
【目的】
- 普通体（「です・ます」なし）で哲学的深みを出す
- THINK-RAGで謙虚な気づきとフィラーを使う

【構成要素】
- 個人ストーリーRAG（承/転）
- Core思想RAG
- 偉人RAG（触媒）
- THINK-RAG ← NEW!

【例】
「実装してみて気づくのは、構造の力です。
 うーん、僕もこの部分はずっと考えてるんです。
 数字を追いかけるんじゃなくて、構造を理解する。
 構造が先か、思想が先か──その往復が面白いです。
 【触媒】古い指標を捨てて、新しい価値基準に合わせていく。
 たしかに、知識を得ただけでは動けません。【counterpoint】
 でも、それだけで結果は変わります。」

【THINK-RAGの使い方】
✅ フィラー（「うーん、」「たしかに、」）を1〜2箇所に挿入
✅ 謙虚な気づき（「僕もこの部分は〜」「その往復が面白い」）を使う
❌ 「私は〜してきた」「〜を信じてきた」（実績アピール）はNG

【偉人RAG（触媒）の使い方】
✅ your_voice_paraphrase を自然に溶かし込む
✅ counterpoint を1文挿入（フィラーと組み合わせOK）
❌ 偉人の名前も原文（quote）も絶対に出さない

【文字数配分】
- 偉人RAG: 約20〜50文字（全体の2〜5%）
- THINK-RAG: 約80〜100文字（全体の10%）
```

### 4️⃣ Body2（250文字）: 具体的な実装方法を示す

```plaintext
【目的】
- ブログ記事の実装詳細
- 「では、具体的にどう実装するか。」で始める
- 客観的な情報提供に徹する

【例】
「では、具体的にどう実装するか。
 実は、シンプルです。
 実装の要は二つあります。
 一つ目はドキュメントの整備です。
 もう一つは検証ループです。
 これで無駄がなくなります。
 コストも、時間も削減できます。
 構造が正しければ、結果は出ます。」

【❌ 避ける表現】
- 「要は二つ。」（断片的）

【✅ 推奨表現】
- 「要は二つあります。一つ目は〜、もう一つは〜です。」（完結、自然）
```

### 5️⃣ CTA（100文字）: 視聴者への問いかけで締める

```plaintext
【目的】
- 視聴者への問いかけで行動を促す
- 個人的な主張は一切しない

【例】
「あなたが次に構造化すべき業務は何ですか？
 どこから始めますか？
 詳しい実装方法は、概要欄のリンクから確認してください。
 それでは、また次の動画で。」

【❌ 避ける表現】
- 「感情は語りません。構造で示します。」（偉そう、実績アピール）
- 「私はAIを使わない。」（実績アピール）
- 「構造で示す。それだけのこと。」（唐突、文脈なし）

【✅ 推奨表現】
- 「あなたが次に〜すべきは何ですか？」（視聴者への語りかけ、完結、謙虚）
```

## 🎯 「格」の残し方

個人的な主張は削除しましたが、**「格」はトーンで残しています**：

1. **静かな決意** → 文章のリズムと余白で表現
2. **哲学的深み** → Twistセクションで普通体を使用
3. **抑えた熱** → 完結した文章、淡々とした語り口
4. **構造へのこだわり** → 「この構造を見ると〜」という表現
5. **謙虚な探求心** → THINK-RAGで「僕もこの部分は〜」

つまり、**内容の深さと語り口のトーンで「格」を保ちつつ、個人的な主張は一切しない**という戦略です。

### RAGテーブル構成

#### 1. personal_story_rag

```sql
CREATE TABLE personal_story_rag (
  id SERIAL PRIMARY KEY,
  story_arc TEXT NOT NULL,              -- 起/承/転/結
  section_title TEXT NOT NULL,          -- 「序章」「覚醒」「孤独」「覚悟」
  content TEXT NOT NULL,                -- 原文
  core_theme TEXT NOT NULL,             -- 核心テーマ
  emotion TEXT NOT NULL,                -- 感情トーン
  voice_direction TEXT,                 -- 「低く、抑えた熱」等
  rhythm_note TEXT,                     -- 「1文ごとに間を取る」等
  use_cases TEXT[],                     -- intro/body/twist/cta
  keywords TEXT[],                      -- 「構造」「孤独」「静かな」等
  embedding_text TEXT NOT NULL,         -- ベクトル生成用
  embedding VECTOR(3072),               -- 3072次元ベクトル
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 2. catalyst_rag

```sql
CREATE TABLE catalyst_rag (
  id SERIAL PRIMARY KEY,
  person TEXT NOT NULL,                 -- 偉人名
  quote TEXT NOT NULL,                  -- 原文
  theme TEXT NOT NULL,                  -- テーマ
  subthemes TEXT[],                     -- サブテーマ
  emotion TEXT,                         -- 感情名（日本語）
  emotion_code TEXT NOT NULL,           -- 感情コード
  your_voice_paraphrase TEXT NOT NULL,  -- Kenjiの声で言い換え
  core_message TEXT NOT NULL,           -- 核心メッセージ
  use_cases TEXT[],                     -- intro/twist/climax/cta
  insertion_style TEXT,                 -- direct_quote/allusion/paraphrase
  counterpoint TEXT,                    -- 反論・バランス
  source_url TEXT,                      -- 出典URL
  verify_status TEXT DEFAULT 'unverified', -- verified/credible/unverified
  lang TEXT,                            -- en/ja
  embedding_text TEXT NOT NULL,         -- ベクトル生成用
  embedding VECTOR(3072),               -- 3072次元ベクトル
  created_at TIMESTAMP DEFAULT NOW()
);
```
