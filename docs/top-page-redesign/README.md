## 🏗️ アーキテクチャ概要

### 三層構造
```
┌─────────────────────────────────────┐
│ Layer 1: THE SCREAM（叫びの層）     │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│ • Hero + THE SWITCH                  │
│ • 感情訴求・世界観没入               │
│ • Fragment IDなし（動的コンテンツ）  │
└─────────────────────────────────────┘
         ↓ スクロール
┌─────────────────────────────────────┐
│ Layer 2: THE LOGIC（論理の層）       │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│ • Problem → Solution → Price        │
│ • Bento Grid レイアウト              │
│ • 助成金・価格ロジック               │
└─────────────────────────────────────┘
         ↓ スクロール
┌─────────────────────────────────────┐
│ Layer 3: THE BLUEPRINT（設計図の層） │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│ • Philosophy（原田賢治セクション）   │
│ • Services Architecture（23 Fragment IDs）│
│ • Footer + Trust Signals             │
└─────────────────────────────────────┘
```

### Fragment ID保持戦略
現在の**23個のベクトルリンク**は完全保持：
- 12個のサービス Fragment ID
- 8個のFAQ Fragment ID
- 3個のAIサイトセクション Fragment ID

これらは**折りたたみ式「Services Blueprint」セクション**に配置し、AI検索エンジンからの引用可能性を維持。
