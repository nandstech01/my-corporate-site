## 📐 提案アーキテクチャ

### 自動ブログ改善エンジン（5フェーズループ）

```
┌─────────────────────────────────────────────────────────────┐
│               自動ブログ改善エンジン                              │
│          (Auto Blog Improvement Engine)                     │
└─────────────────────────────────────────────────────────────┘

1️⃣ 【生成】RAGブログ記事自動生成
   ├─ Triple RAG (Company + Trend + YouTube)
   ├─ 複数パラメータセット (A/B/Cテスト)
   └─ Fragment ID / 構造化データ自動生成
              ↓
2️⃣ 【追跡】パフォーマンスメトリクス収集
   ├─ Googleアナリティクス: PV、滞在時間、直帰率
   ├─ AI引用追跡: ChatGPT/Perplexity引用回数
   ├─ ベクトル類似度: RAG検索ランキング
   └─ エンゲージメント: SNSシェア、コメント
              ↓
3️⃣ 【評価】Django RAG評価API → MLflow記録
   ├─ Recall@k: RAG検索精度
   ├─ MRR (Mean Reciprocal Rank)
   ├─ NDCG: 検索結果品質
   ├─ セマンティック整合性スコア
   └─ ビジネスKPI: CV率、問い合わせ数
              ↓
4️⃣ 【学習】パターン分析 & パラメータ最適化
   ├─ 高パフォーマンス記事の特徴抽出
   ├─ 最適RAG比率計算 (Company:Trend:YouTube)
   ├─ 最適文字数・構造分析
   └─ カテゴリ別最適パラメータ学習
              ↓
5️⃣ 【改善】次回生成へフィードバック (自動ループ)
   ├─ 学習済みパラメータで再生成
   ├─ A/Bテスト継続（新パターン vs 最適解）
   └─ 週次/月次で自動最適化
              ↓
        【1. 生成フェーズ】へ戻る（自動ループ）
```

## 🔧 実装コンポーネント

### 1. パフォーマンストラッキングシステム (新規)
- Googleアナリティクス連携
- AI引用数追跡（ChatGPT/Perplexity/Claude）
- RAG検索ランキング計算
- エンゲージメント追跡
- **実装場所**: `backend/rag_api/tracking/`

### 2. A/Bテスト自動実行エンジン (新規)
- 複数パラメータセット自動生成
- 同時並行記事生成（A/B/Cテスト）
- パフォーマンス比較・統計検定
- **実装場所**: `backend/rag_api/ab_testing/`

### 3. パラメータ最適化エンジン (新規)
- 機械学習モデル: RandomForest / XGBoost
- 高パフォーマンス記事の特徴抽出
- 最適RAG比率計算
- カテゴリ別最適パラメータ学習
- **実装場所**: `backend/rag_api/ml_optimization/`

### 4. 自動フィードバックループ (新規)
- Celery週次最適化タスク
- 低パフォーマンス記事の自動再生成
- Grafana/MLflow自動レポート
- **実装場所**: `backend/rag_api/tasks/`

### 5. Grafana/MLflow統合ダッシュボード (拡張)
- パフォーマンス推移グラフ
- A/Bテスト結果比較
- RAG比率最適化推移
- トップ10記事ランキング
- AI引用数推移
- カテゴリ別パフォーマンス

## 📊 新規データモデル（3テーブル）

### 1. `blog_performance_metrics`
- トラフィックメトリクス（PV、滞在時間、直帰率）
- AI引用メトリクス（ChatGPT/Perplexity/Claude）
- RAG検索メトリクス（ランキング、類似度）
- エンゲージメントメトリクス（シェア、コメント）
- ビジネスKPI（CV率、問い合わせ数）
- **総合スコア**: performance_score (0-100)

### 2. `ab_test_experiments`
- 実験ID、実験名
- パラメータセット（A/B/Cパターン）
- 実験結果（勝者パターン、統計的有意性）

### 3. `blog_generation_parameters`
- 記事ID、実験ID
- RAG比率パラメータ（Company/Trend/YouTube）
- 生成パラメータ（文字数、図解設定）
- カテゴリ・クエリ
