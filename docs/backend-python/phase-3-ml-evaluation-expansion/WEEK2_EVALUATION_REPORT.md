## 📌 カテゴリ別詳細分析

### 1. Technical カテゴリ（固有名詞含む、15クエリ）

**最重要カテゴリ**: 固有名詞（製品名、技術用語、人名など）を含むクエリ

| Metric | Baseline | BM25 | Improvement |
|--------|----------|------|-------------|
| **Precision@5** | 0.0923 | 0.1231 | **+33.3%** ✅ |
| **MRR** | 0.2766 | 0.4330 | **+56.5%** ✅ |
| **Recall@20** | 0.5769 | 0.5769 | **+0.0%** ⚠️ |
| **NDCG@20** | 0.3313 | 0.4151 | **+25.3%** ✅ |

**🔍 解釈**:
- ✅ **MRRで劇的改善**（+56.5%）: 最初の正解を見つける精度が大幅向上
- ✅ **NDCG@20で改善**（+25.3%）: 順位を考慮した精度も向上
- ⚠️ **Recall@20は変わらず**（+0.0%）: 上位20件での網羅性は同じ
  - **重要**: これはBM25が**再ランキング**であることと整合
  - BM25は「正解を上位に持ってくる」のは得意だが、「新たに正解を拾う」わけではない
  - Vectorがすでに20件中に正解を含んでいる場合、BM25は順位を改善するのみ

**代表的なクエリ例**:
- `Mike King`: MRR 0.50 → 1.00 (+100%)
- `SAP RAG`: MRR 0.33 → 0.50 (+51.5%)
- `DataRobot`: MRR 0.25 → 0.33 (+32%)

## 🎯 結論

### ✅ BM25を本番採用（確定）

**理由**:
1. **Technical（固有名詞）カテゴリで劇的改善**
   - MRR +56.5%は決定的
   - NDCG@20 +25.3%も十分な改善
2. **General（一般語）カテゴリでは変化なし**
   - 悪化ではなく、横ばい（許容範囲）
   - Vector単独が既に強いことを示唆
3. **Recall@20が変わらないのは期待通り**
   - BM25は再ランキング（順位改善）が本質
   - 網羅性（新規発見）は向上しないのが正常

### 🔬 RRFを科学的に否定（Week 2A実験結果）

**理由**:
- RRF_K ∈ {10, 30, 60} で全て劣後
- BM25が既に強く、融合すると票割れが起きる
- 実験で否定したこと自体が、研究者・エンプラAIエンジニアの正しいプロセス

## 📝 技術的考察

### Recall@20が変わらない理由

**BM25の本質は「再ランキング」**:
1. Vector検索が上位20件を取得
2. BM25がその20件を再ランキング
3. **新たに正解を追加するわけではない**

**図解**:
```
[Vector Search]
  正解A (20位) ─┐
  正解B (15位) ─┤
  ノイズ1 (1位) ─┤
  ノイズ2 (2位) ─┤  → [BM25 Re-rank] →  正解A (1位) ← MRR改善！
  ...             │                    正解B (2位)
  正解C (8位)  ─┘                    正解C (3位)
                                       ノイズ1 (4位)
                                       ...

→ 正解の「順位」は改善（MRR↑, NDCG↑）
→ 正解の「個数」は変わらず（Recall→）
```

**これは期待通りの挙動であり、BM25の設計通り！**

### Q1: 本番では technical / general をクエリ分類して分岐しますか?

**A**:
