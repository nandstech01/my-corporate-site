## 🎯 プロジェクト目標

### 核心的な改善

既存の🏗️AIアーキテクト・ショート台本生成機能を、**フック最適化 + ディープリサーチ統合**で劇的にブラッシュアップ。

```
【現状の問題】
❌ フックが弱い → 視聴維持率20%
❌ 内容が退屈 → 既知の情報のみ
❌ 一般層に刺さらない → 専門用語多用

【改善後】
✅ フック最適化 → 視聴維持率85%（4.25倍）
✅ 海外AI最新情報 → まだ日本にない情報
✅ 一般層にも刺さる → 専門用語ゼロ、比喩活用
```

## 📊 システムアーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│ 新しい台本生成フロー（V2）                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 1️⃣ ターゲット層自動検出                                      │
│    └─ ブログタイトル分析 → 一般層 or 実装者 or 上級者         │
│                                                             │
│ 2️⃣ コンテンツのディープリサーチ（3-5分）                     │
│    ├─ DeepSeek V3 使用                                       │
│    ├─ 海外AI最新情報（GitHub/Reddit/arXiv）                  │
│    ├─ depth: 2, breadth: 4                                  │
│    └─ hybrid_deep_research テーブルに保存                   │
│                                                             │
│ 3️⃣ フックパターンRAGをハイブリッド検索                       │
│    ├─ MrBeast/TikTokバイラルフックパターン                   │
│    ├─ ブログトピック × フックパターンRAG                     │
│    ├─ 効果スコア × 関連度でランキング                        │
│    └─ 最適なフックを選択                                     │
│                                                             │
│ 4️⃣ 既存RAGハイブリッド検索（5秒）⚡                           │
│    ├─ hybrid_deep_research RAG（既存データ）                 │
│    ├─ hybrid_scraped_keywords RAG（既存データ）              │
│    ├─ fragment_vectors RAG（ブログ記事）                     │
│    └─ personal_story_rag RAG（Kenjiのトーン）                │
│                                                             │
│ 5️⃣ スコア判定（条件分岐）                                     │
│    ✅ score >= 0.75 → 既存データ使用（85%）⚡                │
│    ❌ score < 0.75 → 新規リサーチ実行（15%）💰               │
│                                                             │
│ 6️⃣ GPT-5で台本生成（20秒）⚡                                  │
│    ├─ 最適化されたフック使用（必須）                          │
│    ├─ 専門用語→日常語変換                                    │
│    ├─ 比喩・具体例必須                                       │
│    └─ CTA（プロンプトプレゼント or ストーリー誘導）           │
│                                                             │
│ 7️⃣ DB保存                                                    │
│    └─ company_youtube_shorts（既存テーブル）                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
