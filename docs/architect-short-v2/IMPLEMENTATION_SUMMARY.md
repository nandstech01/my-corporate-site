## 🎬 台本生成フロー（V2）

```
┌─────────────────────────────────────────────────────────────┐
│ 1. ユーザーが🏗️ショートボタンをクリック                       │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. API: /api/generate-architect-short-v2                     │
│    └─ ブログ記事情報を受け取る                                │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. ターゲット層自動検出（1秒）                                │
│    └─ 「ホームページ制作 滋賀県」→ 一般層                    │
│    └─ 「ベクトルDB おすすめ」→ 実装者                        │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 既存RAGをハイブリッド検索（5秒）⚡                         │
│    ├─ hybrid_deep_research RAG（既存データ）                 │
│    ├─ hybrid_scraped_keywords RAG（既存データ）              │
│    ├─ fragment_vectors RAG（ブログ記事）                     │
│    └─ personal_story_rag RAG（Kenjiのトーン）                │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. スコア判定（条件分岐）                                     │
│    ✅ score >= 0.75 → 既存データで十分（85%のケース）        │
│    └─ Phase 6へ（高速ルート）⚡                              │
│    ❌ score < 0.75 → 新規リサーチ必要（15%のケース）         │
│    └─ Phase 5-Bへ（低速ルート）💰                            │
└─────────────────────────────────────────────────────────────┘
    ↓ (score >= 0.75の場合: 85%)
┌─────────────────────────────────────────────────────────────┐
│ 6. フックパターンをハイブリッド検索（5秒）⚡                  │
│    ├─ ブログトピック × viral_hook_patterns RAG              │
│    ├─ 効果スコア × 関連度でランキング                        │
│    ├─ 最適パターン選択（例: transformation型）               │
│    └─ 変数抽出 → フック生成                                  │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 7. GPT-5で台本生成（20秒）⚡                                  │
│    ├─ 既存RAGデータを使用                                    │
│    ├─ 最適化されたフック使用（必須）                          │
│    ├─ 専門用語→日常語変換                                    │
│    ├─ 比喩・具体例必須                                       │
│    └─ CTA（プロンプトプレゼント or ストーリー誘導）           │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 8. company_youtube_shorts に保存                             │
│    └─ metadata にフック情報、使用RAG情報を追加               │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 9. 既存の台本管理画面にリダイレクト                           │
│    └─ /admin/youtube-scripts/{scriptId}                     │
└─────────────────────────────────────────────────────────────┘

⚡ 総所要時間: 約30秒（85%のケース - 既存RAG使用）
```

## ⚠️ リスクと対策

### リスク1: 生成時間が長い場合がある（4-7分）

**対策:**
- 85%のケースは既存RAG使用で30秒以内 ⚡
- 15%のケースのみ新規リサーチ（4-7分）💰
- ローディング表示「高品質な台本を生成中...」
- 非同期処理化（バックグラウンド実行）← Phase 8で実装可能

### リスク2: ディープリサーチのコスト

**対策:**
- DeepSeek V3使用（GPT-4より安価）
- キャッシュ活用（同じトピックは再利用）

### リスク3: フックパターンRAGの品質

**対策:**
- 初期データを厳選（MrBeast/TikTok高再生数のみ）
- 効果測定→フィードバックループ
