## 🗂️ V2のDB設計（概要）

### 新規テーブル（V2で追加）

```sql
-- ========================================
-- 階層構造
-- ========================================
rag_documents          -- 記事レベル
  └─ document_id
  └─ title
  └─ type

rag_document_sections  -- セクションレベル
  └─ section_id
  └─ document_id (FK)
  └─ is_part_of (hasPart/isPartOf)
  └─ has_parts

rag_document_fragments -- フラグメントレベル
  └─ fragment_id
  └─ section_id (FK)
  └─ content
  └─ embedding
  └─ is_part_of
  └─ has_parts

-- ========================================
-- エンティティ
-- ========================================
rag_entities          -- エンティティ
  └─ entity_id
  └─ name
  └─ type (person/place/org/tech/concept)
  └─ canonical_name
  └─ aliases
  └─ embedding

rag_fragment_entities -- 関連付け
  └─ fragment_id (FK)
  └─ entity_id (FK)
  └─ relevance_score

rag_entity_relations  -- エンティティ間の関係
  └─ from_entity_id (FK)
  └─ to_entity_id (FK)
  └─ relation_type
  └─ strength
```

詳細は `/docs/architect-short-v2/RAG_DESIGN_DECISION.md` を参照。
