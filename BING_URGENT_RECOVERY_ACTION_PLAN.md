## 🔍 除外理由別の対処法（ステップ4の結果を基に実施）

### ケース1: 「重複コンテンツ」として除外
**原因**: 類似ページが既にインデックス済み

**対処法**:
```typescript
// 各ページに canonical タグを明示
// app/[page]/layout.tsx または page.tsx

export const metadata: Metadata = {
  alternates: {
    canonical: 'https://nands.tech/[page-path]',
  },
};
```

---

### ケース2: 「クロール済み - インデックス未登録」
**原因**: Bingがページを見たが、インデックス価値が低いと判断

**対処法**:
```
1. コンテンツの品質向上
   - 文字数: 最低500文字、理想2000文字以上
   - 独自性: 他ページと差別化

2. 内部リンクの追加
   - トップページから該当ページへのリンク
   - 関連ページ間の相互リンク

3. 構造化データの追加
   - 既に実装済みだが、ページ固有の情報を強化
```

---

### ケース3: 「発見済み - 未クロール」
**原因**: サイトマップには含まれているが、まだクロールされていない

**対処法**:
```
1. URL検査で手動リクエスト（ステップ3参照）
2. 内部リンクを増やす
3. 待つ（通常1-2週間）
```

---

### ケース4: 「noindexタグが設定されている」
**原因**: 意図せず noindex が設定されている

**対処法**:
```
1. 該当ページのソースコードを確認
   - <meta name="robots" content="noindex"> がないか確認

2. Next.jsのmetadataを確認
   export const metadata: Metadata = {
     robots: 'index, follow', // 明示的に指定
   };
```

---

### ケース5: 「robots.txtでブロック」
**原因**: robots.txtで該当パスが Disallow されている

**対処法**:
```
1. public/robots.txt を確認
   - 該当パスが Disallow されていないか確認

2. 現在の設定（確認済み）:
   ✅ Allow: /
   ✅ Disallow: /partner-admin, /referrer-admin, /partners, /admin/partners/
   
   → 問題なし（除外対象は正しい）
```

---

### Q2: 所有権確認ができない
**A**:
```
1. 確認ファイルが正しくアクセス可能か確認:
   curl https://nands.tech/BingSiteAuth.xml

2. 404の場合:
   - ファイルがpublic/に配置されているか確認
   - ファイル名が正確か確認（大文字小文字含む）

3. それでもダメな場合:
   - HTMLメタタグ方式に変更
   - または DNS TXTレコード方式に変更
```

---

### Q3: サイトマップが送信できない
**A**:
```
1. サイトマップが正常に生成されているか確認:
   curl https://nands.tech/sitemap.xml | head -20

2. 404エラーの場合:
   - app/sitemap.ts が存在するか確認
   - ビルドエラーがないか確認

3. XMLフォーマットエラーの場合:
   - サイトマップバリデーターで確認:
     https://www.xml-sitemaps.com/validate-xml-sitemap.html
```

---

```
   - サイトマップ再送信
   - 主要ページの手動リクエスト再実施

4. それでもダメな場合:
   - Bingサポートに問い合わせ

```
